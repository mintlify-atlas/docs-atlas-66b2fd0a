This page documents all TypeScript types exported by VirtualizorJS. All types are fully typed and exported from the main package entry point.

## Configuration Types

### VirtualizorConfig

Configuration object for initializing the Virtualizor client.

```typescript
interface VirtualizorConfig {
  host: string;
  apiKey: string;
  apiPass?: string;
  port?: number;
  https?: boolean;
  rejectUnauthorized?: boolean;
  timeout?: number;
  debug?: boolean;
}
```

<ResponseField name="host" type="string" required>
  Virtualizor server hostname or IP address (without protocol or port)
</ResponseField>

<ResponseField name="apiKey" type="string" required>
  API key for authentication
</ResponseField>

<ResponseField name="apiPass" type="string">
  API password for authentication (defaults to empty string if not provided)
</ResponseField>

<ResponseField name="port" type="number">
  Server port (defaults to 4085 for HTTPS, 4084 for HTTP)
</ResponseField>

<ResponseField name="https" type="boolean">
  Use HTTPS for connections (defaults to true)
</ResponseField>

<ResponseField name="rejectUnauthorized" type="boolean">
  Reject self-signed SSL certificates (defaults to true)
</ResponseField>

<ResponseField name="timeout" type="number">
  Request timeout in milliseconds (defaults to 30000)
</ResponseField>

<ResponseField name="debug" type="boolean">
  Enable debug logging (defaults to false)
</ResponseField>

### VirtType

Union type representing supported virtualization technologies.

```typescript
type VirtType =
  | 'kvm'
  | 'xen'
  | 'openvz'
  | 'lxc'
  | 'proxmox'
  | 'virtuozzo'
  | 'xcp'
  | 'hyperv';
```

Supported virtualization types:
- `kvm` - Kernel-based Virtual Machine
- `xen` - Xen hypervisor
- `openvz` - OpenVZ containers
- `lxc` - Linux Containers
- `proxmox` - Proxmox VE
- `virtuozzo` - Virtuozzo containers
- `xcp` - XenServer/XCP-ng
- `hyperv` - Microsoft Hyper-V

## Error Types

### VirtualizorApiError

Custom error class thrown when API requests fail.

```typescript
class VirtualizorApiError extends Error {
  readonly code: number;
  
  constructor(message: string, code: number);
}
```

<ResponseField name="message" type="string" required>
  Error message from the API
</ResponseField>

<ResponseField name="code" type="number" required>
  Virtualizor error code
</ResponseField>

<ResponseField name="name" type="string" required>
  Always set to 'VirtualizorApiError'
</ResponseField>

This error is thrown automatically by the SDK when the Virtualizor API returns an error response. You can catch it to handle API errors specifically:

```typescript
import { VirtualizorApiError } from 'virtualizorjs';

try {
  await client.vps.create({ /* ... */ });
} catch (error) {
  if (error instanceof VirtualizorApiError) {
    console.error(`API Error ${error.code}: ${error.message}`);
  }
}
```
