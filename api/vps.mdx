---
title: 'VPS'
description: 'Manage virtual private servers'
---

The `vps` resource provides comprehensive methods for managing virtual private servers, including lifecycle operations, configuration management, and monitoring.

## Overview

Access the VPS resource through the client:

```typescript
const vps = client.vps;
```

## Methods

### Information & Retrieval

#### list()

Retrieve all VPS instances with optional filtering.

```typescript
list(filters?: ListVPSParams): Promise<Record<string, VPS>>
```

<ParamField path="filters" type="ListVPSParams" optional>
  Optional filtering parameters
  
  <Expandable title="properties">
    <ParamField path="user" type="string | number" optional>
      Filter by user ID
    </ParamField>
    <ParamField path="vpsid" type="string | number" optional>
      Filter by VPS ID
    </ParamField>
    <ParamField path="vpsname" type="string" optional>
      Filter by VPS name
    </ParamField>
    <ParamField path="vpsip" type="string" optional>
      Filter by VPS IP address
    </ParamField>
    <ParamField path="vpshostname" type="string" optional>
      Filter by hostname
    </ParamField>
    <ParamField path="vsstatus" type="string" optional>
      Filter by status (e.g., "online", "offline")
    </ParamField>
    <ParamField path="vstype" type="VirtType" optional>
      Filter by virtualization type (kvm, xen, openvz, lxc, proxmox, virtuozzo, xcp, hyperv)
    </ParamField>
    <ParamField path="serid" type="number" optional>
      Filter by server ID
    </ParamField>
    <ParamField path="plid" type="number" optional>
      Filter by plan ID
    </ParamField>
    <ParamField path="bpid" type="number" optional>
      Filter by billing plan ID
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="result" type="Record<string, VPS>">
  A dictionary of VPS instances keyed by VPS ID
  
  <Expandable title="VPS properties">
    <ResponseField name="vpsid" type="string" required>
      Unique VPS identifier
    </ResponseField>
    <ResponseField name="hostname" type="string" required>
      VPS hostname
    </ResponseField>
    <ResponseField name="os_name" type="string" required>
      Operating system name
    </ResponseField>
    <ResponseField name="ram" type="string" required>
      Allocated RAM in MB
    </ResponseField>
    <ResponseField name="bandwidth" type="string" required>
      Monthly bandwidth limit in GB
    </ResponseField>
    <ResponseField name="vps_name" type="string" optional>
      Custom VPS name
    </ResponseField>
    <ResponseField name="uuid" type="string" optional>
      VPS UUID
    </ResponseField>
    <ResponseField name="serid" type="string | number" optional>
      Server ID where VPS is hosted
    </ResponseField>
    <ResponseField name="uid" type="string | number" optional>
      User ID who owns the VPS
    </ResponseField>
    <ResponseField name="plid" type="string | number" optional>
      Plan ID
    </ResponseField>
    <ResponseField name="suspended" type="string | number" optional>
      Suspension status (0 = active, 1 = suspended)
    </ResponseField>
    <ResponseField name="ips" type="Record<string, string>" optional>
      IP addresses assigned to the VPS
    </ResponseField>
    <ResponseField name="cores" type="string | number" optional>
      Number of CPU cores
    </ResponseField>
    <ResponseField name="space" type="string | number" optional>
      Disk space in GB
    </ResponseField>
  </Expandable>
</ResponseField>

**Example:**

```typescript
// List all VPS instances
const allVps = await client.vps.list();
console.log(Object.keys(allVps).length); // Number of VPS instances

// Filter by user
const userVps = await client.vps.list({ user: '123' });

// Filter by virtualization type
const kvmVps = await client.vps.list({ vstype: 'kvm' });

// Filter by status
const onlineVps = await client.vps.list({ vsstatus: 'online' });
```

### Lifecycle Operations

<Note>
  All lifecycle methods return `AsyncTaskResult` and trigger asynchronous operations. Use the [tasks](/api/tasks) resource to monitor completion.
</Note>

#### create()

Create a new VPS instance.

```typescript
create(params: CreateVPSParams): Promise<AsyncTaskResult>
```

<ParamField path="params" type="CreateVPSParams" required>
  VPS creation parameters
  
  <Expandable title="properties">
    <ParamField path="hostname" type="string" required>
      VPS hostname
    </ParamField>
    <ParamField path="rootpass" type="string" required>
      Root/administrator password
    </ParamField>
    <ParamField path="osid" type="number" required>
      Operating system template ID
    </ParamField>
    <ParamField path="plid" type="number" optional>
      Plan ID to use for resource allocation
    </ParamField>
    <ParamField path="user_email" type="string" optional>
      Email address for VPS owner
    </ParamField>
    <ParamField path="ips" type="number" optional>
      Number of IPv4 addresses to assign
    </ParamField>
    <ParamField path="ips_int" type="number" optional>
      Number of internal IPs
    </ParamField>
    <ParamField path="space" type="number" optional>
      Disk space in GB
    </ParamField>
    <ParamField path="ram" type="number" optional>
      RAM in MB
    </ParamField>
    <ParamField path="burst" type="number" optional>
      Burst RAM in MB
    </ParamField>
    <ParamField path="bandwidth" type="number" optional>
      Monthly bandwidth in GB
    </ParamField>
    <ParamField path="cpu" type="number" optional>
      Number of CPU cores
    </ParamField>
    <ParamField path="cpu_percent" type="number" optional>
      CPU percentage limit
    </ParamField>
    <ParamField path="virt" type="VirtType" optional>
      Virtualization type (kvm, xen, openvz, lxc, proxmox, virtuozzo, xcp, hyperv)
    </ParamField>
    <ParamField path="serid" type="number" optional>
      Server ID to create VPS on
    </ParamField>
    <ParamField path="node_select" type="0 | 1" optional>
      Automatic node selection (0 = manual, 1 = automatic)
    </ParamField>
    <ParamField path="recipe" type="number" optional>
      Post-installation recipe ID
    </ParamField>
    <ParamField path="sshkey" type="string" optional>
      SSH public key to inject
    </ParamField>
    <ParamField path="nopassword" type="0 | 1" optional>
      Disable password authentication (requires sshkey)
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="result" type="AsyncTaskResult">
  <Expandable title="properties">
    <ResponseField name="taskid" type="string" optional>
      Task ID for monitoring the creation progress
    </ResponseField>
    <ResponseField name="done" type="1" optional>
      Set to 1 if operation completed immediately
    </ResponseField>
  </Expandable>
</ResponseField>

**Example:**

```typescript
const result = await client.vps.create({
  hostname: 'web-server-01.example.com',
  rootpass: 'SecureP@ssw0rd!',
  osid: 123,
  plid: 5,
  ram: 2048,
  space: 50,
  bandwidth: 1000,
  cpu: 2,
});

if (result.taskid) {
  // Wait for creation to complete
  const task = await client.tasks.wait(result.taskid);
  console.log('VPS created successfully!');
}
```

#### rebuild()

Reinstall the operating system on a VPS.

```typescript
rebuild(vpsId: string, params: RebuildVPSParams): Promise<AsyncTaskResult>
```

<ParamField path="vpsId" type="string" required>
  The VPS ID to rebuild
</ParamField>

<ParamField path="params" type="RebuildVPSParams" required>
  Rebuild parameters
  
  <Expandable title="properties">
    <ParamField path="osid" type="number" required>
      OS template ID to rebuild with
    </ParamField>

    </ParamField>
    <ParamField path="conf" type="1" required>
      Must be set to 1 to confirm the rebuild
    </ParamField>
    <ParamField path="format_primary" type="0 | 1" optional>
      Format primary disk (0 = no, 1 = yes); defaults to 0
    </ParamField>
    <ParamField path="eu_send_rebuild_email" type="0 | 1" optional>
      Send rebuild notification email (0 = no, 1 = yes)
    </ParamField>
    <ParamField path="recipe" type="number" optional>
      Recipe ID to apply post-rebuild
    </ParamField>
    <ParamField path="sshkey" type="string" optional>
      SSH public key to inject
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="result" type="AsyncTaskResult">
  Task result (see create() for property documentation)
</ResponseField>

**Example:**

```typescript
const result = await client.vps.rebuild('12345', {
  osid: 456,
  newpass: 'NewSecureP@ssw0rd!',
  conf: 1,
  format_primary: 1,
});

if (result.taskid) {
  await client.tasks.wait(result.taskid);
  console.log('VPS rebuilt successfully');
}
```

<Warning>
  Rebuilding will erase all data on the VPS. Ensure you have backups before proceeding.
</Warning>

#### migrate()

Migrate a VPS to a different server.

```typescript
migrate(vpsId: string, params: MigrateVPSParams): Promise<AsyncTaskResult>
```

<ParamField path="vpsId" type="string" required>
  The VPS ID to migrate
</ParamField>

<ParamField path="params" type="MigrateVPSParams" required>
  Migration parameters
  
  <Expandable title="properties">
    <ParamField path="serid" type="number" required>
      Destination server ID
    </ParamField>
    <ParamField path="from_ip" type="string" required>
      Source server IP address
    </ParamField>
    <ParamField path="from_pass" type="string" required>
      Source server API/root password
    </ParamField>
    <ParamField path="ignore_if_vdfconflict" type="0 | 1" optional>
      0 = ignore VDF conflicts, 1 = error on conflict
    </ParamField>
    <ParamField path="disable_gzip" type="0 | 1" optional>
      0 = use gzip compression, 1 = disable gzip
    </ParamField>
    <ParamField path="online" type="0 | 1" optional>
      0 = offline migration, 1 = live migration
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="result" type="AsyncTaskResult">
  Task result (see create() for property documentation)
</ResponseField>

**Example:**

```typescript
const result = await client.vps.migrate('12345', {
  serid: 5,
  from_ip: '192.168.1.100',
  from_pass: 'source-server-password',
  online: 1, // Live migration
});

if (result.taskid) {
  await client.tasks.wait(result.taskid, {
    timeoutMs: 600000, // 10 minutes for migration
  });
  console.log('VPS migrated successfully');
}
```

<Info>
  Live migration (online = 1) minimizes downtime but requires compatible configurations on both servers.
</Info>

## Type Definitions

### VirtType

Supported virtualization types:

```typescript
type VirtType =
  | 'kvm'
  | 'xen'
  | 'openvz'
  | 'lxc'
  | 'proxmox'
  | 'virtuozzo'
  | 'xcp'
  | 'hyperv';
```

### AsyncTaskResult

Returned by all asynchronous operations:

```typescript
interface AsyncTaskResult {
  taskid?: string;  // Task ID for monitoring progress
  done?: 1;         // Set to 1 if operation completed immediately
}
```

See the [Tasks API](/api/tasks) for monitoring asynchronous operations.
